# LLM 软件设计流水线（01–10）

> 本仓库包含一套 **面向 B 端 / 行业系统的软件设计提示词流水线**。
> 目标不是“辅助思考”，而是 **系统性降低返工率、控制设计质量、冻结可交付物**。

这不是聊天提示词合集，而是一条 **有顺序、有门禁、有回退规则的设计管线**。

---

## 一、整体定位

**输入**：模糊、非结构化的业务想法
**输出**：

* 可冻结的模块与数据结构
* 可执行的业务流程（含异常）
* 可渲染的 UI 页面规格
* 可直接用于生图的 UI Prompt

**明确不包含**：

* 技术实现
* 代码
* 测试执行

---

## 二、标准执行顺序（强制）

**任何项目必须严格按以下顺序执行，不得跳步、合并、倒序。**

```
01 问题澄清与反问
02 问题拆解
03 需求风险扫描        ← Gate ①
04 决策与方案对比
05 模块边界与职责冲突审计
06 数据模型与状态完整性校验  ← Gate ②
07 关键流程异常枚举
08 需求分析与页面规格生成
09 页面-流程-权限一致性校验  ← Gate ③
10 UI Prompt 渲染
```

---

## 三、三道“硬 Gate”（质量门禁）

### Gate ①：需求风险 Gate（03）

**作用**：
拦下目标不清、边界失控、隐含假设过多的需求。

**触发条件**（任一即阻断）：

* 存在「阻断级风险」
* 成功标准不可验证
* 核心对象定义不清

**失败回退**：
→ 回到 `01 问题澄清与反问` 或 `02 问题拆解`

---

### Gate ②：数据与状态 Gate（06）

**作用**：
拦下“流程跑得通，但数据一定会炸”的设计。

**触发条件**：

* 核心实体无明确状态
* 状态过载或不可回收
* 页面 / 流程依赖隐式状态

**失败回退**：
→ 回到 `05 模块边界审计` 或 `04 决策与方案对比`

---

### Gate ③：UI 一致性 Gate（09）

**作用**：
拦下“页面画出来了，但系统跑不通”的假设计。

**触发条件**：

* 流程步骤无页面入口
* 页面操作无权限主体
* 异常流程无 UI 承载方式

**失败回退**：
→ 回到 `08 页面规格生成` 或 `07 关键流程异常枚举`

---

## 四、各提示词的职责边界（简表）

| 编号 | 名称           | 做什么         | 明确不做什么 |
| -- | ------------ | ----------- | ------ |
| 01 | 问题澄清与反问      | 确认是否是真问题    | 不给方案   |
| 02 | 问题拆解         | 拆成可行动原子     | 不冻结结构  |
| 03 | 需求风险扫描       | 标记不可控风险     | 不解决风险  |
| 04 | 决策与方案对比      | 单选决策        | 不头脑风暴  |
| 05 | 模块边界审计       | 找职责冲突       | 不重构架构  |
| 06 | 数据与状态校验      | 查状态缺陷       | 不写表结构  |
| 07 | 流程异常枚举       | 补失败路径       | 不改主流程  |
| 08 | 页面规格生成       | 冻结页面说明      | 不做视觉   |
| 09 | 页面-流程-权限校验   | 找断点         | 不新增页面  |
| 10 | UI Prompt 渲染 | 生成生图 Prompt | 不做设计决策 |

---

## 五、关键顺序约束（必须遵守）

1. **03 不得重复 01 已澄清的问题**

   * 只能引用，不得重新提问

2. **02 不得冻结模块 / 数据 / 状态**

   * 否则会削弱 05 / 06 的价值

3. **06 必须先于 07**

   * 先稳状态，再枚举异常

4. **09 未通过，10 禁止执行**

   * UI 渲染不是“试试看”，是最终输出

---

## 六、推荐使用方式（务实）

### 单人使用（你现在的模式）

* 严格顺序执行
* Gate 失败立即回退
* 不在同一轮对话里跳步骤

### 团队使用（建议）

* 01–04：产品 / 方案阶段
* 05–07：架构与流程评审阶段
* 08–09：设计冻结阶段
* 10：视觉交付阶段

---

## 七、一句话原则（请记住）

> **能被 Gate 拦下的设计，
> 一定比上线后返工要便宜。**
